# Orderly 模板与自定义系统

本文档描述 Orderly 基于 JSON 的经营模板（Business Template）与自定义系统。模板系统让小微商家在不改代码的前提下，按自身经营方式调整自定义字段、页面配置与订单工作流。

## 目标

- 个性化无需改 schema：每个实体通过单一 `CustomFieldsJson` 字段承载个性化数据。
- 可移植：模板以 JSON 作为导入 / 导出的序列化格式，便于分享与复用。
- 中性内置：系统只提供一个行业无关的内置模板，不提供任何行业专属内置模板。

## 模板服务

模板由 `IBusinessTemplateService` 管理。用户对模板执行创建、编辑、激活、克隆、导入、导出操作时，服务统一以 JSON 作为导入 / 导出序列化格式完成操作。

### 导入校验

- 若 JSON 导入载荷未通过 schema 校验，或引用了未定义的通用领域模型实体类型，导入被拒绝。
- 拒绝时返回指明具体校验失败原因的错误，且所有既有模板保持不变。
- JSON 往返一致：对任意合法模板，导出为 JSON 再导入，得到等价模板，保留页面配置、工作流配置与自定义字段定义。

## 内置模板：DefaultCommerce

- 系统提供且仅提供一个内置经营模板。
- 其内部稳定键为 `DefaultCommerce`，面向用户的简体中文显示名为 `默认经营模板`。
- 不提供任何行业专属内置模板。
- 当某工作区没有显式激活的模板时，应用以内部稳定键 `DefaultCommerce` 标识的内置模板作为该工作区的激活模板。
- 在面向用户的界面中只显示简体中文显示名 `默认经营模板`，不显示英文键名。开发文档与内部代码可以使用内部稳定键 `DefaultCommerce`。

## 自定义字段

自定义字段由 `ICustomFieldService` 管理：

- 每个 `CustomFieldDefinition` 关联唯一一个通用领域模型实体类型。
- 每个实体类型支持 0 到 100 个 `CustomFieldDefinition` 条目；尝试为某实体类型新增第 101 个条目会被拒绝。
- 自定义字段的取值持久化到对应实体的 `CustomFieldsJson` 列中，并在保存边界做格式良好性校验。

## 页面配置

经营模板支持页面配置，由以下部分组成：

- 指标卡的显示 / 隐藏状态；
- 表格列的显示 / 隐藏状态；
- 默认排序；
- 默认单位；
- 默认币种；
- 默认订单流程。

其中每个显示 / 隐藏状态都恰好取“显示”或“隐藏”两个值之一。

## 工作流配置

经营模板支持工作流配置，定义覆盖三个独立阶段维度——`OrderSalesStage`、`OrderPaymentStage`、`OrderFulfillmentStage`——的默认工作流：

- 工作流可定义“复合流转”，每个流转一次更新一个、两个或全部三个阶段维度。
- 工作流为三个维度各自指定一个初始阶段值。
- 当请求的阶段流转被当前工作流允许时，订单服务只把该流转命名的维度更新到目标值。
- 当请求的阶段流转不被允许时，三个阶段维度全部保持不变，不产生部分更新，并返回“流转不被允许”的错误结果。

## 与界面的关系

页面配置与工作流配置驱动 WPF 界面在各经营页面上的呈现与可用动作，但界面始终只显示中性的简体中文文案；内置模板对用户只呈现 `默认经营模板`。模板系统不引入任何行业专属字段、文案或受限词。
