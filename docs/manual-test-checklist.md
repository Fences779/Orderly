# 手测清单

> Legacy / Historical Reference
>
> 本文是旧小程序手测清单，只供历史追溯，不代表当前 `main` 的发布前验收基线。
>
> 当前主线发布前请改看 `docs/RELEASE_CHECK.md`、`docs/QA_AUTOMATION.md` 与 `tools/qa/README.md`。

## 1. 极速新建 -> 查看 deal 详情

1. 首页点击「极速新建」。
2. 填客户名、来源平台、一句话需求。
3. 保存。
4. 确认跳转到 deal 详情。
5. 检查时间轴有 `deal_create` 留痕。

## 2. 粘贴板录入 -> 解析 -> 确认入库

1. 复制一段咨询文本，例如：`小林：想定制一条简约珍珠项链，预算300左右，这周要，vx: lin_vx88`。
2. 首页进入「粘贴板录入」。
3. 点击解析。
4. 确认可信度、品类、预算、时间和风险词。
5. 进入草稿确认页。
6. 选择已有客户或新建客户。
7. 修改字段后确认入库。
8. 检查 capture 状态变成 confirmed，并跳转 deal 详情。

## 3. OCR 录入 -> 修正文案 -> 确认入库

1. 首页进入「OCR 录入」。
2. 选择任意图片。
3. mock OCR 返回示例文本。
4. 修改 OCR 文本。
5. 解析并进入确认页。
6. 确认入库。
7. 检查正式 customer/deal 只有确认后才产生。

## 4. 创建报价 -> 发送报价 -> 自动生成跟进任务

1. 进入一个 `quote_preparing` 的 deal。
2. 点击「报价」。
3. 从 SKU 选择报价项。
4. 调整定制加价、工费、运费、优惠、定金。
5. 点击「发送报价」。
6. 检查报价详情可复制对外文本。
7. 返回 deal 详情，检查 `latestQuoteId` 关联。
8. 检查 deal 状态自动变为 `quote_sent`。
9. 检查产生 `quote_no_reply` 跟进任务。

## 5. deal 状态推进到发货 / 收货 / 完成

1. 进入看板。
2. 点击卡片「推进」。
3. 按状态机逐步推进：`scheduled -> in_production -> ready_to_ship -> shipped -> received -> completed`。
4. 检查非法跳转不会出现。
5. 到 `received` 后检查自动生成收货关怀任务。
6. 到 `completed` 后检查自动生成复购任务。

## 6. 运行 followup scan -> 查看任务变化

1. 首页点击「扫描」或设置页点击「扫描跟进任务」。
2. 进入跟进任务页。
3. 检查任务按 `priorityScore DESC`、`dueAt ASC` 排序。
4. 重复扫描一次，确认没有重复生成同一规则任务。

## 7. 模板替换变量 -> 复制文案

1. 进入模板管理。
2. 新建模板，内容包含 `{{customerName}}`、`{{productName}}`。
3. 保存后查看预览。
4. 点击复制。
5. 检查复制文本已替换变量。

## 8. 跟进任务执行

1. 进入跟进任务页。
2. 点击复制文案。
3. 点击完成。
4. 再找一条任务点击延期。
5. 再找一条任务点击跳过。
6. 检查任务状态变化和 activity log。

## 9. 客户详情 -> 快速创建复购 deal

1. 进入客户列表。
2. 打开一个客户详情。
3. 修改偏好、避雷点、风险备注并保存。
4. 点击「复购新建」。
5. 检查新 deal 关联同一 customer。

## 10. 统计页数据变化

1. 打开统计页。
2. 切换今日、近7天、近30天。
3. 新建 deal 后刷新，新增咨询增加。
4. 发送报价后刷新，已报价增加。
5. 推进到 `scheduled` 后刷新，成交数/成交率变化。
6. 标记 lost 后刷新，流失数变化。

## 回归重点

- `dealStage` 只能通过状态入口变更。
- OCR 失败时仍能手动输入继续。
- capture 未确认前不会写 customer/deal。
- 报价不写死在 deal 表。
- 重复扫描跟进任务不会脏建。
- 中文文本显示正常，无乱码。
